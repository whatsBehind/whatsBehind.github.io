    grep #

    Introduction to grep #

    grep stands for “Global Regular Expression Print.” It is a powerful command-line utility used for searching plain-text data for lines that match a regular expression. grep is widely used for searching specific patterns within files and outputting the matching lines.

    Basic Syntax # 

    grep [options] pattern [file...]
    • pattern: The string or regular expression to search for.
    • [file...]: The file(s) to search. If no files are specified, grep searches the input provided from the standard input.

    Common Options #

    • -i: Ignore case distinctions in the pattern.
    • -v: Invert the match, showing lines that do not match the pattern.
    • -r or -R: Recursively search directories.
    • -n: Prefix each line of output with the line number within its input file.
    • -l: Print only the names of files containing matches.
    • -c: Print only a count of matching lines per file.
    • -E: Use extended regular expressions.
    • -o: Only show the matched words

    Practices #

    Let’s have some hands-on practices for grep.

    Set up Practicing Env #

    Navigate to a directory that you want to practice grep

    cd ~/practics/mac-os/terminal-commands/grep

    Write contents to file practice.txt

    cat <<EOL > practice.txt
practice
practices
Practices
Learn with practices
I am practicing grep command
EOL

    Create a new directory and copy practice.txt to the nested directory

    mkdir nested_dir
cp practice.txt nested_dir

    Verify that you have the practice env set up

    tree .

    The file structure should look like below

    .
├── nested_dir
│   └── practice.txt
└── practice.txt

2 directories, 2 files

    Make Your Hands Dirty #

    • Basic Search

      Searches for “practice” in file practice.txt, and prints the matching lines

      grep practice practice.txt

      The matching words are case-sensitive

      practice
practices
Learn with practices

    • Case-Insensitive Search

      Searches for “practice” in practice.txt, ignoring case.

      grep -i practice practice.txt

      Word “Practice” with capital “P” is matched

      practice
practices
Practices
Learn with practices

    • Recursive Search

      Search for practice in all files under current directory, including nested directory

      grep -r practice .

      Output

      ./nested_dir/practice.txt:practice
./nested_dir/practice.txt:practices
./nested_dir/practice.txt:Learn with practices
./practice.txt:practice
./practice.txt:practices
./practice.txt:Learn with practices

    • Count Matches

      Prints the number of lines containing ‘pattern’ in practice.txt

      grep -c practice practice.txt

      Output

      3

    • Display Line Numbers

      Searches for “practice” in practice.txt and displays the line numbers of matching lines

      grep -n practice practice.txt

      Output

      1:practice
2:practices
4:Learn with practices

    • Search for Exact Word

      Searches for the exact word “practice” in practice.txt

      grep -w practice practice.txt

      Only word practice is exactly matched

      practice

    • Invert Match

      Prints lines that do not match “practice” in practice.txt

      grep -v practice practice.txt

      Output

      Practices
I am practicing grep command

    • Display Only Filenames

      Prints the names of files containing ‘pattern’ within the current directory

      grep -lr practice .

      Output

      ./nested_dir/practice.txt
./practice.txt

    • Using Extended RegEx

      Searches for “practice” and “practicing” in file practice.txt using extended regular expressions

      grep -E "practice|practicing" practice.txt

      Output

      practice
practices
Learn with practices
I am practicing grep command

    • Only Show Matched Parts

      Only shows the matched word “practice” and prints line number

      grep -on practice practice.txt

      Output

      1:practice
2:practice
4:practice
