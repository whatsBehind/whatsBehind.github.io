    Concepts #

    __name__ #

    __name__ is a built-in variable (attribute) of a module. It is being used to indicate if a module is being run directly or being imported into other modules

    • Value when module is running directly: When you run a Python module directly, __name__ of this module is set to __main__

    • Value when module is imported: When the module is imported into others modules, __name__ is set to the module name (file name w/o .py extension). For example, a Python module my_module.py is imported, then its __name__ is set to my_module when it is imported

    Example #

    Assume you have two Python files main.py and my_module.py, and main.py is the running script and imports my_module.py

    • Directory structure

      src
├── main.py
└── my_module.py

    • my_module.py

      # my_module.py file
print(f"__name__ in my_module file: {__name__}")

    • main.py

      # main.py file
import my_module

print(f"__name__ in main file: {__name__}")

    Run python3 main.py in src directory, and you see logs in the terminal

    __name__ in my_module file: my_module
__name__ in main file: __main__

    Python Path (sys.path) #

    The sys.path in Python is a list of strings that specifies the search path for modules. When you import a module, the Python interpreter searches for the module in each directory listed in sys.path in order, from the first to the last. The search process stops as soon as the interpreter finds a module that matches the import statement, and then it attempts to load the module.

    Here’s what typically comprises sys.path:

    • The directory of the running script: The first entry in sys.path is the path to the directory containing the running script.

    • PYTHONPATH Directories: If the PYTHONPATH environment variable is set, its contents are added to sys.path. PYTHONPATH is a list of directories separated by os.pathsep (which is ; on Windows and : on Unix/Linux) where Python looks for modules to import.

    • Standard Library Directories: These are the directories that contain the Python standard library modules. The location of these directories depends on the Python installation.

    • Site-packages Directory: This directory contains third-party modules installed using package managers like pip. There can be multiple site-packages directories if Python is configured with virtual environments or if there are user-specific installations (using pip install --user).

    Built-in Modules #

    Built-in modules are an integral part of Python and are automatically loaded with the Python interpreter, providing foundational functionality that Python code can utilize. They don’t have specific files like standard Python modules (.py files) on the disk, so they are not under directories in sys.path

    • To see all built-in modules of your Python interpreters:

      import sys
print(sys.builtin_module_names)

      You will see a list of built-in modules printed in the terminal. Some common built-in modules:

      ('sys', 'time', 'math')

    • To see origin of a module:

      import importlib.util

module_name = "sys"
module_spec = importlib.util.find_spec(module_name)
print(f"Origin of module [{module_name}]: [{module_spec.origin}]")
      • stdout 
        Origin of module [sys]: [built-in]

      Origin of non-built-in modules, like site-package modules, is the directory containing the module. For example

      Origin of module [langchain]: [/home/<user_name>/.local/lib/python3.10/site-packages/langchain/__init__.py]

    sys.modules #

    sys.modules is a dictionary in Python. It acts as a cache of all modules that have already been imported during the runtime of a program. This dictionary maps module names (as strings) to module objects, ensuring that each module is imported only once, even if it is referenced in multiple import statements across the program.

    Caching Mechanism #

    When you import a module, Python first checks sys.modules to see if the module is already loaded. If it is, Python uses the existing module object from sys.modules instead of reloading the module from file.

    Assume you have two Python files. When running python3 module_cache.py, you should only see “I am imported” get printed once.

    src
├── module_cache.py
└── my_module.py

    • my_module.py

      # my_module.py
print("I am imported")

    • module_cache.py

      # module_cache.py
import my_module
import my_module

    Module Compilation #

    • Source Code Compilation

      When a Python module is imported or a Python script is executed, the Python interpreter first compiles the source code (.py files) into bytecode (.pyc files). Bytecode is a low-level, platform-independent representation of the source code that can be executed by the Python Virtual Machine (PVM).

    • Bytecode Execution

      After compilation, the bytecode is directly executed by the Python interpreter.

    • In-Memory Compilation

      The bytecode for the running script is compiled in memory and is not automatically saved to disk.

    • On-Disk Cache

      The bytecode of imported modules are cached on disk (__pycache__ directory) for further use. The __pycache__ directory is typically located in the same directory as the .py source file it corresponds to

    Namespace #

    A namespace in Python is essentially a mapping from names to objects (dict[str, Any]). It’s a container of all variables and their values (primitive type) or references (objects) in current scope

    Type of Namespaces #

    • Built-in namespace: Contains names provided by Python itself, such as built-in functions (print, len) and exception names.

    • Global Namespace: Specific to a module, this contains names from various imported modules and variables defined at the level of the current module. Each module has its own global namespace.

    • Local Namespace: Specific to a function or method invocation, this contains names defined in the function. These names only exist while the function is executing.

    Examples #

    Below example explains the main purpose of namespace: Prevent naming conflicts by isolating names in separate scopes

    string = "global namespace"

def my_func():
    string = "local namespace of my_func"
    print(string)

    def nested_func():
        string = "local namespace of nested_func"
        print(string)

    nested_func()

my_func()
print(string)

    • Initially, string is defined in global namespace of current module

    • When function my_func() is invoked, a local namespace is created for it during its execution. A new string variable is defined in the local namespace

    • Within my_func(), nested_func() is defined and invoked. Another local namespace is created and nested within the local namespace of my_func(). Within the nested local namespace, a new string is defined

    Output

    local namespace of my_func
local namespace of nested_func
global namespace

    Execution Process #

    File and File Structure #

    • File structure

      src
├── main.py
└── my_module.py

    • Files

      # my_module.py

print("I am my_module.")

def greeting():
    print("Hello World!")

      # main.py
import my_module

my_module.greeting()

    Executing python3 main.py #

    • Python interpreter (python3) initializes

    • Python constructs sys.path, including:

      • The directory of the script being run (src in this case)

      • Standard library directories

      • Directories in PYTHONPATH

      • Site-packages directories

    • Python compiles and executes main.py

    • Python loads my_module.py

      • First Python searches the module in sys.modules cache

      • Because it is first time importing my_module, so the module is not in sys.modules cache. Then Python starts to search directories in sys.path list

      • Python found my_module.py in the same directory containing the running script main.py and loads the file

      • If there is no cache file in __pycache__ under src or the source code of my_module.py has been modified since last compilation, Python compiles the source code into bytecode and stores it in __pycache__

      • After compiling my_module, Python starts to execute it and print I am my_module in the terminal

    • Python adds my_module to global namespace of main.py

    • Python continues executing main.py

      • Invokes greeting() function from my_module, and prints Hello World! in the terminal
