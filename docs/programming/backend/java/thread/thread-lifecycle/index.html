    Six Java Thread States #

    • NEW State:

      • When you create an instance of a Thread class (or a class that extends Thread), the thread is in the New state.
      • At this point, the thread is not yet running.
      • Example: Thread t = new Thread();

    • RUNNABLE State:

      • When you invoke the start() method, the thread moves to the RUNNABLE state
      • In a typical implementation, a Java thread in the RUNNABLE state corresponds to an OS thread that is eligible for running
      • A RUNNABLE state means the thread is
        • either running on the CPU
        • or waiting for Scheduler to allocate CPU resource

    • BLOCKED State:

      • A thread enters the BLOCKED state when it tries to acquire a monitor lock (intrinsic lock) of an object but another thread already holds that lock. This commonly occurs in synchronized blocks or methods.

    • WAITING State:

      • A thread is in the WAITING state when it is waiting for another thread to perform a specific action. Unlike the BLOCKED state, where a thread is waiting to acquire a lock, in the WAITING state, a thread is waiting for another thread’s action without any lock competition
      • How WAITING state occurs
        • Object.wait() WITHOUT timeout
        • Thread.join() WITHOUT timeout

    • TIMED_WAITING State:

      • A thread is in this state when it is waiting for another thread to perform an action for up to a specified waiting time. After the time expires, the thread will automatically return to the RUNNABLE state if it’s not blocked by other means
      • How TIMED_WAITING state occurs
        • Thread.sleep(long millis)
        • Object.wait WITH timeout
        • Thread.join WITH timeout

    • TERMINATED State:

      • A thread enters the TERMINATED state when it completes the execution of its run() method or if an exception occurs.
      • Once terminated, a thread cannot be restarted.

      @Slf4j
public class SixStates {
    public static void main(String[] args) {
        Thread newThread = new Thread(() -> {
            log.debug("Start a new thread");
        }, "NEW-thread");

        Thread runnableThread = new Thread(() -> {
            while(true) {

            }
        }, "RUNNABLE-thread");
        runnableThread.start();

        Thread timedWaitingThread = new Thread(() -> {
            try {
                Thread.sleep(1_000_000);
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
        }, "TIMED_WAITING-thread");
        timedWaitingThread.start();

        Thread waitingThread = new Thread(() -> {
            try {
                synchronized (SixStates.class) {
                    runnableThread.join();
                }
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
        }, "WAITING-thread");
        waitingThread.start();

        Thread blockedThread = new Thread(() -> {
            synchronized (SixStates.class) {
                log.debug("BLOCKED thread");
            }
        }, "BLOCKED-thread");
        blockedThread.start();

        Thread terminatedThread = new Thread(() -> {

        }, "TERMINATED-thread");
        terminatedThread.start();

        log.debug("{}", newThread.getState());
        log.debug("{}", runnableThread.getState());
        log.debug("{}", timedWaitingThread.getState());
        log.debug("{}", waitingThread.getState());
        log.debug("{}", blockedThread.getState());
        log.debug("{}", terminatedThread.getState());
    }
}
