    üß≠ Lesson 1: Set Up Kubernetes Locally and Run Your First Pod (macOS Edition) #

    üî¢ Goal #

    By the end of this lesson, you’ll be able to:

    • Install a local Kubernetes cluster using Minikube on macOS
    • Understand the basic components behind Kubernetes operations
    • Deploy and explore your very first Pod using the command line

    üõ†Ô∏è Step 1: Install Required Tools (macOS) #

    We‚Äôll install three essential tools:

    • Docker: Runs containers locally on your machine.
    • Minikube: Sets up a simple Kubernetes cluster on your laptop.
    • kubectl: A command-line tool to control Kubernetes resources.

    1. Install Docker Desktop #

    Docker provides the container runtime that Kubernetes will use under the hood.

    brew install --cask docker

    After installation, open Docker Desktop from your Applications folder and ensure it is running. You should see the Docker whale icon in the macOS menu bar.

    2. Install Minikube #

    Minikube creates a single-node Kubernetes cluster that runs locally.

    brew install minikube

    Verify installation:

    minikube version

    3. Install kubectl #

    kubectl is your main interface to interact with Kubernetes clusters.

    brew install kubectl

    Verify installation:

    kubectl version --client

    üöÄ Step 2: Start Your Kubernetes Cluster #

    Minikube uses Docker to simulate a cluster with one node. Before continuing, make sure Docker Desktop is running.

    Once Docker is ready, start your Kubernetes cluster:

    minikube start

    This command will:

    • Spin up a virtual Kubernetes node
    • Launch the control plane (the core of Kubernetes that manages apps)
    • Configure networking so Kubernetes can route traffic to your apps

    To confirm everything is working:

    kubectl get nodes

    Expected output:

    NAME       STATUS   ROLES           AGE   VERSION
minikube   Ready    control-plane   1m    v1.xx.x

    ‚úÖ Congratulations ‚Äî you now have a running Kubernetes cluster!

    üè¢ Step 3: Run Your First Pod #

    In Kubernetes, a Pod is the smallest deployable unit. It wraps one or more containers. Let‚Äôs create a Pod that runs an nginx web server:

    kubectl run my-first-pod --image=nginx

    This command tells Kubernetes:

    • “Create a Pod named my-first-pod”
    • “Use the official nginx container image”

    To verify that your Pod is running:

    kubectl get pods

    Example output:

    NAME            READY   STATUS    RESTARTS   AGE
my-first-pod    1/1     Running   0          20s

    üîé Step 4: Explore the Pod #

    Let‚Äôs dig into the Pod to see what‚Äôs happening inside.

    View Pod Details #

    This command provides detailed info about the Pod:

    kubectl describe pod my-first-pod

    It includes:

    • What node it’s running on
    • Which container image is used
    • Event logs like when the image was pulled and started

    Open a Shell Inside the Pod #

    To really understand what‚Äôs running inside your Pod, you can open a shell session into its container. Think of it like opening a terminal on a remote machine ‚Äî you can inspect files, check logs, or run commands.

    kubectl exec -it my-first-pod -- /bin/bash

    Once inside, try a few basic commands:

    ls              # Lists files in the container
curl localhost  # Tests if the nginx server is running
exit            # Exits the shell and returns to your local terminal

    ‚ùå Step 5: Clean Up (Optional) #

    When you’re finished, it’s a good idea to clean up your environment:

    Delete the Pod:

    kubectl delete pod my-first-pod

    Stop the Minikube cluster:

    minikube stop

    Keeping your system clean will make future exercises easier to manage.

    üîÑ Recap #

    In this lesson, you:

    • Installed Docker, Minikube, and kubectl on macOS
    • Started a local Kubernetes cluster
    • Created and explored your first Kubernetes Pod
    • Opened a shell inside the container to see it in action

    These are foundational steps that will support everything you do in Kubernetes going forward.
