    üß± Project Part 1: Set Up Backend API with FastAPI #

    üéØ Goal #

    In this part of the project, you’ll create a FastAPI application, containerize it with a Dockerfile, and orchestrate it using Docker Compose. This backend service will serve as the foundation of your fullstack application.

    üóÇÔ∏è Project Structure Overview #

    Start with the following folder structure:

    docker-fullstack-app/

    ‚úÖ Step 1: Create the FastAPI App #

    Inside the backend/ folder:

    app.py # 

    from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"message": "Hello from FastAPI"}

    requirements.txt # 

    fastapi
uvicorn

    ‚úÖ Step 2: Write the Dockerfile #

    In the backend/ folder:

    Dockerfile # 

    FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["uvicorn", "app:app", "--host", "0.0.0.0", "--port", "8000"]

    This Dockerfile installs the dependencies and runs your FastAPI app with Uvicorn.

    ‚úÖ Step 3: Add .dockerignore #

    Still inside the backend/ folder:

    .dockerignore # 

    __pycache__/
*.pyc
*.pyo
*.log

    This helps reduce build context size and keeps the image clean.

    ‚úÖ Step 4: Define the Docker Compose File #

    In the root directory docker-fullstack-app/, create the following:

    docker-compose.yml # 

    version: "3.9"
services:
  backend:
    build:
      context: ./backend
    ports:
      - "8000:8000"

    This file defines a backend service built from the Dockerfile and exposes it on port 8000.

    ‚úÖ Step 5: Build and Run the Service #

    From the root project directory:

    docker-compose up --build

    Open your browser and go to http://localhost:8000. You should see:

    {"message":"Hello from FastAPI"}

    üîç Step 6: Inspect the Running Container #

    To open a shell session inside the running container:

    docker exec -it docker-fullstack-app-backend-1 sh

    Check the contents of the app directory:

    ls /app

    You should see your source files (app.py, etc.) inside /app.

    ‚úÖ Congratulations! Your backend API is now running inside a container. You’re ready to move on to Project Part 2: Add PostgreSQL and Connect to FastAPI.
