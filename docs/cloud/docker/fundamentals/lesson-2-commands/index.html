    üîÑ 1. Lifecycle of a Docker Container #

    A typical container lifecycle includes the following steps:

    1. Create ‚Äì using docker create or docker run
    2. Start ‚Äì using docker start
    3. Run ‚Äì the container is now live
    4. Stop ‚Äì using docker stop
    5. Remove ‚Äì using docker rm

    üîß 2. Common Docker Commands #

    Command Description
    docker run Create and start a container
    docker ps List running containers
    docker ps -a List all containers (running, stopped, exited)
    docker stop <id> Stop a running container
    docker start <id> Start a stopped container
    docker rm <id> Remove a stopped container
    docker rm -f <id> Force-remove a running container (stop and delete)
    docker logs <id> View logs generated by a container
    docker exec -it <id> bash Open a bash shell inside a container

    üí° ps stands for process status ‚Äî a familiar command in Linux. In Docker, docker ps lists running containers.

    üíª Hands-On Practice #

    ‚úÖ Step 1: Run an Nginx Web Server #

    We‚Äôll begin with Nginx ‚Äî a lightweight, high-performance web server ‚Äî to explore practical container operations.

    docker run -d -p 8080:80 --name webserver nginx
    • -d: Run the container in detached (background) mode
    • -p 8080:80: Map port 8080 on your machine to port 80 inside the container
    • --name webserver: Assign the container a name for easy reference
    • nginx: Use the official Nginx image

    Now open your browser and go to http://localhost:8080 ‚Äî you should see the Nginx welcome page.

    ‚úÖ Step 2: Inspect and Manage the Container #

    List running containers:

    docker ps

    List all containers, including stopped and exited ones:

    docker ps -a

    This also displays each container‚Äôs status, such as:

    • Up 1 minute
    • Exited (0) (stopped successfully)
    • Exited (1) (stopped with an error)

    Stop the container:

    docker stop webserver

    Start the container again:

    docker start webserver

    View container logs:

    docker logs webserver

    Access an interactive shell session inside the container:

    docker exec -it webserver bash

    ‚úÖ Step 3: Remove the Container #

    To remove a container, it must be stopped first:

    docker stop webserver

    If the container is still running and you want to force-remove it:

    docker rm -f webserver

    ‚ö†Ô∏è This forcibly stops and deletes the container. Use with caution.

    Verify the container is gone:
