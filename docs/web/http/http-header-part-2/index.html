    üìò Mastering Content-Type and Accept Headers with FastAPI #

    This lesson focuses on two essential HTTP headers: Content-Type and Accept. These headers play a critical role in defining the data format being sent or expected in API communication. Understanding how to use them correctly is key to building robust, flexible APIs.

    üîπ Content-Type: Declaring the Format of Sent Data #

    • Direction: Request and Response
    • Purpose: Informs the recipient how to parse the message body.

    Common Values

    Value Description
    application/json JSON format (default in most APIs)
    text/plain Plain text
    application/x-www-form-urlencoded URL-encoded form data
    multipart/form-data Used for file uploads with forms

    Example:

    Content-Type: application/json

    This tells the server that the request body is formatted as JSON.

    üî∏ Accept: Indicating the Desired Response Format #

    • Direction: Request only
    • Purpose: Tells the server which content types the client is willing to accept.

    Common Values

    Value Description
    application/json Requests a JSON response
    text/plain Requests a plain text response
    */* Accepts any response format (default)

    Example:

    Accept: application/json

    This tells the server the client prefers a response formatted as JSON.

    üõ†Ô∏è Hands-On Practice with FastAPI #

    The following FastAPI app demonstrates how to inspect the Content-Type and Accept headers and respond accordingly.

    üìÑ Start Your Server #

    Change directory to 03-http-headers and create a new Python file content_type_and_accept.py:

    cd 03-http-headers
touch content_type_and_accept.py

    Paste in the following Python code:

    from fastapi import FastAPI, Header, Request
from fastapi.responses import JSONResponse, PlainTextResponse
from typing import Optional

app = FastAPI()

@app.post("/echo")
async def echo_content_type(
    request: Request,
    content_type: Optional[str] = Header(None)
):
    body = await request.body()
    return {
        "received_content_type": content_type,
        "raw_body": body.decode()
    }

@app.get("/greet")
async def greet(
    accept: Optional[str] = Header("application/json")
):
    if "text/plain" in accept:
        return PlainTextResponse("Hello, plain world!")
    return JSONResponse({"message": "Hello, JSON world!"})

    Use Uvicorn to start your FastAPI app:

    uvicorn content_type_and_accept:app --reload

    üß™ Try These curl Commands #

    Run the following curl command in another terminal:

    # Send plain text body with Content-Type header
curl -X POST http://localhost:8000/echo -H "Content-Type: text/plain" -d "hello"

# Request plain text response using Accept header
curl -H "Accept: text/plain" http://localhost:8000/greet

# Request JSON response using Accept header
curl -H "Accept: application/json" http://localhost:8000/greet

    You should see response:

    {"received_content_type":"text/plain","raw_body":"hello"}
Hello, plain world!
{"message":"Hello, JSON world!"}
