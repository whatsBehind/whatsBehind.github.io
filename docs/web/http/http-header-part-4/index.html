    📘 Lesson: HTTP Cache Validation – ETag and Last-Modified with FastAPI #

    In the previous lesson, we learned how to control whether and how content is cached using Cache-Control. In this lesson, we focus on cache validation, which allows the server to determine whether a client’s cached response is still valid. If so, it can return a lightweight 304 Not Modified instead of resending the full resource.

    We’ll cover how ETag and Last-Modified headers work, how clients use their corresponding validation headers (If-None-Match and If-Modified-Since), and how servers determine which one takes precedence. We’ll also walk through a complete FastAPI example and test it using the browser console.

    🔍 Why Validate Cached Content? #

    Cache validation reduces unnecessary data transfer and accelerates response times. Rather than re-downloading content that hasn’t changed, the client asks the server:

    “Has this changed since I last received it?”

    If the server determines it has not changed, it returns:

    HTTP/1.1 304 Not Modified

    This saves bandwidth and improves speed.

    🔗 What Are ETag and Last-Modified? #

    When responding to a request, a server can include one or both of the following headers to help the client validate the content later:

    🔹 ETag (Entity Tag)

    • A unique identifier for the content (usually a hash or version string).
    • If the content changes, the ETag value changes.
    • The client stores this value and uses it in the If-None-Match header on future requests.
    • Example: 
      ETag: "abc123"

    🔹 Last-Modified

    • A timestamp indicating when the resource was last changed.
    • Sent in the Last-Modified header.
    • The client uses this value in a future If-Modified-Since header.
    • Example: 
      Last-Modified: Wed, 10 Apr 2024 12:00:00 GMT

    When sending a request, the client can include one or both following headers to allow the server to validate the cache:

    🔸 If-None-Match

    • Sent by the client to check if the server’s current ETag differs from what the client has.
    • If the values match, the server returns 304 Not Modified.
    • Example: 
      If-None-Match: "abc123"

    🔸 If-Modified-Since

    • Sent by the client to check if the resource has changed since the given timestamp.
    • If it has not, the server responds with 304 Not Modified.
    • Example: 
      If-Modified-Since: Wed, 10 Apr 2024 12:00:00 GMT

    🔄 How Do These Headers Work Together? #

    Header Role Sent By Paired With
    ETag Version ID Server If-None-Match
    If-None-Match Validator Client ETag
    Last-Modified Timestamp Server If-Modified-Since
    If-Modified-Since Validator Client Last-Modified

    🧭 Precedence Rules #

    • If both If-None-Match and If-Modified-Since are present:
      1. The server must evaluate If-None-Match first.
      2. If it matches → return 304 Not Modified
      3. If it does not match, evaluate If-Modified-Since
      4. If the timestamp is still valid → return 304 Not Modified
      5. Otherwise → return new content (200 OK)

    💡 ETag is generally more precise than Last-Modified and preferred for fine-grained cache validation.

    🛠️ FastAPI Hands-On Example #

    In this example, we dynamically generate a response string that includes a counter updated every 10 seconds. The server sets both an ETag (based on content hash) and a Last-Modified timestamp. The client can use either or both to validate the cached content.

    Change directory to 03-http-headers and create a new Python file cache-validation.py:

    cd 03-http-headers
touch cache-validation.py

    Paste in the following Python code:

    from fastapi import FastAPI, Request, Response
from fastapi.responses import JSONResponse
from datetime import datetime
import hashlib

app = FastAPI()

@app.get("/validate")
def validate_cache(request: Request):
    # Calculate counter and last_modified from current time
    now = datetime.utcnow()
    epoch_seconds = int(now.timestamp())
    counter = epoch_seconds // 100

    content = f"This is the cached data #{counter}"
    last_modified = datetime.utcfromtimestamp(counter * 100)
    etag = hashlib.md5(content.encode()).hexdigest()

    # 1. ETag validation
    if_none_match = request.headers.get("if-none-match")
    if if_none_match == etag:
        return Response(
            status_code=304,
            headers={"X-Cache-Validated-By": "etag"}
        )

    # 2. Last-Modified validation
    if_modified_since = request.headers.get("if-modified-since")
    if if_modified_since:
        since = datetime.strptime(if_modified_since, "%a, %d %b %Y %H:%M:%S GMT")
        if last_modified <= since:
            return Response(
                status_code=304,
                headers={"X-Cache-Validated-By": "last-modified"}
            )

    headers = {
        "ETag": etag,
        "Last-Modified": last_modified.strftime("%a, %d %b %Y %H:%M:%S GMT"),
        "Cache-Control": "public, max-age=0"
    }
    return JSONResponse({ "data": content }, headers=headers)

    🧪 Test It Out Using the Browser Console #

    You can test cache validation directly in the browser using Chrome DevTools:

    1. Open Chrome DevTools (F12)
    2. Go to the Console tab

    Step 1: Make an Initial Request

    Run below command in the Console:

    fetch("http://localhost:8000/validate").then(res => {
    console.log("ETag:", res.headers.get("etag"));
    console.log("Last-Modified:", res.headers.get("last-modified"));
    console.log("Validated By:", res.headers.get("X-Cache-Validated-By"));
    const status = res.status;
    console.log("Response Status Code:", status);
    if (res.status === 200) {
        res.json().then(data => {
            console.log("Response Content:", data);
        });
    }
});

    Console logs:

    ETag: 8fc1ceef992cc55a62591b537d1ec7fa
Last-Modified: Mon, 14 Apr 2025 01:30:00 GMT
Validated By: etag
Response Status Code: 200
Response Content: {data: 'This is the cached data #17445942'}

    The server will:

    • Return 200 status code
    • Return content hash in ETag header
    • Return content modified timestamp in Last-Modified header

    Step 2: Validate with If-None-Match

    Substitute <your-etag-value> in below command with ETag from the response in last step:

    fetch("http://localhost:8000/validate", {
    headers: {
        "If-None-Match": "<your-etag-value>"
    }
}).then(res => {
    console.log("ETag:", res.headers.get("etag"));
    console.log("Last-Modified:", res.headers.get("last-modified"));
    console.log("Validated By:", res.headers.get("X-Cache-Validated-By"));
    const status = res.status;
    console.log("Response Status Code:", status);
    if (res.status === 200) {
        res.json().then(data => {
            console.log("Response Content:", data);
        });
    }
});

    You should see console logs as below. If the server returns 200, it means that the content has been updated. Rerun same command using ETag value in latest 200 response.

    ETag: null
Last-Modified: null
Validated By: etag
Response Status Code:  304

    The server will:

    • Return 304 status code
    • Indicate the cache is validated by ETag

    Step 3: Validate with If-Modified-Since

    Substitute <your-last-modified-date> in below command with Last-Modified from the response in last step:

    fetch("http://localhost:8000/validate", {
    headers: {
        "If-Modified-Since": "<your-last-modified-date>"
    }
}).then(res => {
    console.log("ETag:", res.headers.get("etag"));
    console.log("Last-Modified:", res.headers.get("last-modified"));
    console.log("Validated By:", res.headers.get("X-Cache-Validated-By"));
    const status = res.status;
    console.log("Response Status Code:", status);
    if (res.status === 200) {
        res.json().then(data => {
            console.log("Response Content:", data);
        });
    }
});

    Console logs:

    ETag: null
Last-Modified: null
Validated By: last-modified
Response Status Code:  304

    The server will:

    • Return 304 status code
    • Indicate the cache is validated by Last-Modified

    Step 4: Use Both Headers Together

    fetch("http://localhost:8000/validate", {
    headers: {
        "If-None-Match": "<your-etag-value>",
        "If-Modified-Since": "<your-last-modified-date>"
    }
}).then(res => {
    console.log("ETag:", res.headers.get("etag"));
    console.log("Last-Modified:", res.headers.get("last-modified"));
    console.log("Validated By:", res.headers.get("X-Cache-Validated-By"));
    const status = res.status;
    console.log("Response Status Code:", status);
    if (res.status === 200) {
        res.json().then(data => {
            console.log("Response Content:", data);
        });
    }
});

    ETag: null
Last-Modified: null
Validated By: etag
Response Status Code:  304

    The server will:

    • Return 304 if the ETag matches
    • If not, check timestamp
    • If still valid, return 304
    • Otherwise, return updated content (200 OK)
