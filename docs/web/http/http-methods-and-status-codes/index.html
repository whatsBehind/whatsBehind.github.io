    üìò HTTP Methods & Status Codes #

    Understanding HTTP methods and status codes is essential for designing and debugging web APIs. This lesson explores the most commonly used HTTP methods and how servers communicate the results of requests using standardized status codes.

    üîß Common HTTP Methods #

    HTTP methods (also called verbs) define the action the client wants the server to perform. Here are the most commonly used methods:

    Method Purpose Idempotent Safe
    GET Retrieve data ‚úÖ ‚úÖ
    POST Submit data to create resource ‚ùå ‚ùå
    PUT Replace a resource ‚úÖ ‚ùå
    PATCH Modify part of a resource ‚ùå ‚ùå
    DELETE Remove a resource ‚úÖ ‚ùå

    ‚úÖ Idempotent means the request can be made multiple times with the same result.

    ‚úÖ Safe means the request does not change server state.

    üö¶ HTTP Status Codes #

    HTTP responses include status codes to indicate the outcome of a request. These codes are grouped into categories:

    Code Range Meaning Examples
    1xx Informational 100 Continue
    2xx Success 200 OK, 201 Created, 204 No Content
    3xx Redirection 301 Moved Permanently, 304 Not Modified
    4xx Client Error 400 Bad Request, 404 Not Found
    5xx Server Error 500 Internal Server Error

    üîç Notable 2xx Status Codes #

    • 200 OK: A generic success response. Typically, includes a response body.
    • 201 Created: Indicates that a new resource has been created, commonly used after a successful POST request. May include a Location header with the URL of the newly created resource.
    • 204 No Content: Indicates success but no content is returned in the response body. Often used for DELETE or silent PUT requests.

    üõ†Ô∏è Hands-On Practice with FastAPI #

    In this exercise, you’ll implement endpoints for each HTTP method and return the appropriate status code.

    üìÑ Create or Update main.py #

    Create a new subdirectory 02-http-methods-status-codes and a new Python file main.py:

    mkdir 02-http-methods-status-codes
cd 02-http-methods-status-codes
touch main.py

    Paste in the following Python code:

    from fastapi import FastAPI
from fastapi.responses import JSONResponse
from pydantic import BaseModel

app = FastAPI()

# In-memory store
items = {}

class Item(BaseModel):
    name: str
    description: str

@app.get("/items/{item_id}", status_code=200)
def read_item(item_id: str):
    if item_id not in items:
        return JSONResponse(status_code=404, content={"error": "Item not found"})
    return items[item_id]

@app.post("/items/{item_id}", status_code=201)
def create_item(item_id: str, item: Item):
    items[item_id] = item.dict()
    return items[item_id]

@app.put("/items/{item_id}", status_code=200)
def replace_item(item_id: str, item: Item):
    items[item_id] = item.dict()
    return items[item_id]

@app.patch("/items/{item_id}", status_code=200)
def update_description(item_id: str, item: Item):
    if item_id in items:
        items[item_id]["description"] = item.description
    return items.get(item_id, {"error": "Item not found"})

@app.delete("/items/{item_id}", status_code=204)
def delete_item(item_id: str):
    if item_id in items:
        del items[item_id]
    return
