    TLS Handshake: Server Hello #

    After a client sends the ClientHello message to initiate a secure connection, the server responds with ServerHello. This message finalizes the parameters needed to establish an encrypted session, and it’s the server’s way of saying:

    “Here‚Äôs what I‚Äôve selected from your offers. Let‚Äôs proceed securely.”

    In this post, we‚Äôll break down the ServerHello message, how to view it, and what its key components mean.

    Step 1: Capture ServerHello Using OpenSSL #

    To view the TLS handshake, including ServerHello, use:

    openssl s_client -connect www.google.com:443 -servername www.google.com -msg

    Once connected, look for:

    <<< TLS 1.3 Handshake [length ****], ServerHello

    To analyze the hex data, first store it in a file called server_hello.hex:

    01 00 01 2d 03 03 a1 c6 ed 3f 78 88 21 0c 32 40 ...

    Now, use Python to parse it. Create file tls-parser.py:

    from scapy.all import *
load_layer("tls")

with open("server_hello.hex", "r") as file:
    hex_str = file.read().replace(r"\s+", "")

    pkt = TLSServerHello(bytes.fromhex(hex_str))
    pkt.show()

    Run the Python script to view the ServerClient message:

    python3 tls-parser.py

    Step 2: Key Contents of ServerHello (TLS 1.3) #

    Here‚Äôs a real parsed example and what each field means:

    Field Purpose
    Version The version of TLS chosen by the server (usually TLS 1.3)
    Random Bytes Server-generated random number, used in key generation
    Session ID Optional field used for session resumption (32 bytes here)
    Cipher Suite The encryption algorithm selected by the server (e.g., CHACHA20_POLY1305_SHA256)
    Extensions Additional fields, described below

    Extensions in ServerHello

    TLS 1.3 simplifies and minimizes the extensions in ServerHello. Here’s what we typically see:

    • üîê Key Share

      • Contains the server’s public key and the group (e.g., x25519) used to generate it.
      • Used with the client‚Äôs key to derive the shared symmetric key for encryption.

    • üî¢ Supported Versions

      • Indicates that the server selected TLS 1.3.
      • This confirms the TLS version chosen from the client‚Äôs supported list.

    Summary #

    The ServerHello is the server‚Äôs response to the client‚Äôs handshake offer. It:

    • Selects the final TLS version, cipher, and key parameters
    • Sends the server’s key share (group and key) so that both sides can compute a shared encryption key
    • Confirms the protocol version with extensions like supported_versions

    Together with the ClientHello, this message lays the foundation for encrypted HTTPS communication using a symmetric key agreed upon securely during the handshake.
