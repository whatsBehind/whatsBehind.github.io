    üìù Lesson 1: Kafka Overview and Local Setup #

    üöÄ What You‚Äôll Learn #

    In this lesson, you’ll:

    • Understand Kafka’s core components.
    • Set up a Kafka cluster locally using Docker Compose.
    • Install Kafka CLI tools on your machine.
    • Create a topic and send/consume messages from the command line.

    üöß Kafka Key Concepts #

    Concept Description
    Broker A Kafka server that stores and serves data.
    Topic A logical channel to which messages are published and from which consumers subscribe.
    Partition A topic is split into partitions to support horizontal scaling and parallel processing.
    Producer A client application that sends (publishes) messages to a Kafka topic.
    Consumer A client application that reads (subscribes to) messages from a Kafka topic.
    Consumer Group A group of consumers that work together to consume a topic‚Äôs data in parallel.
    Offset A unique identifier for a message within a partition; consumers track offsets to know which messages have been processed.

    üìÖ Step-by-Step Setup #

    ‚úÖ 1. Create a Project Folder # 

    mkdir kafka-project && cd kafka-project

    ‚úÖ 2. Add a Docker Compose File #

    Create a file named docker-compose.yml:

    version: '3'
services:
  zookeeper:
    image: confluentinc/cp-zookeeper:7.5.0
    environment:
      ZOOKEEPER_CLIENT_PORT: 2181
      ZOOKEEPER_TICK_TIME: 2000

  kafka:
    image: confluentinc/cp-kafka:7.5.0
    depends_on:
      - zookeeper
    ports:
      - "9092:9092"
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://localhost:9092
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1

    üîç What Is Zookeeper? #

    Kafka uses Zookeeper to manage internal metadata and coordination tasks like:

    • Tracking active brokers
    • Electing a controller broker
    • Managing topic configurations

    Note: Kafka is moving away from Zookeeper in favor of KRaft mode, but Zookeeper is still common in many setups today.

    üîç What Is KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR? #

    Kafka uses a special internal topic (__consumer_offsets) to store consumer progress. The variable:

    KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1

    means Kafka stores one copy of that topic (no replication). This is fine for local testing but would be set to 3 or more in production for fault tolerance.

    ‚úÖ 3. Start Kafka # 

    docker-compose up -d
docker ps  # check containers

    ‚úÖ 4. Install Kafka CLI on Your Host #

    macOS: # 

    brew install kafka

    üß∞ Send and Receive Messages from Host #

    Create a topic: # 

    kafka-topics --create \
  --bootstrap-server localhost:9092 \
  --topic test-topic \
  --partitions 1 \
  --replication-factor 1

    List topics: # 

    kafka-topics --list --bootstrap-server localhost:9092

    Send a message: # 

    kafka-console-producer --bootstrap-server localhost:9092 --topic test-topic

    Type something like:

    Hello Kafka!

    Receive the message: # 

    kafka-console-consumer --bootstrap-server localhost:9092 \
  --topic test-topic --from-beginning

    You should see your message printed in the terminal. üéâ

    üìä Wrap-Up #

    You now have:

    • A working Kafka environment with Docker Compose
    • Kafka CLI installed
    • Sent and consumed your first Kafka message
